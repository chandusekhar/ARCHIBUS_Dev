This package contains manager interfaces and classes for implementing flexible workflows.

The main interfaces are the {@link com.archibus.eventhandler.steps.StatusManager} and {@link com.archibus.eventhandler.steps.StepManager} with base class implementations which can be extended to implement activity specific status and step manager classes.

Domain developers can create custom Steps implementing the {@link com.archibus.eventhandler.steps.Step} interface or extending the base class {@link com.archibus.eventhandler.steps.StepImpl}.
Steps demanding a response can extend the abstract {@link com.archibus.eventhandler.steps.Action} class or extend the {@link com.archibus.eventhandler.steps.Acceptance} class.

The {@link com.archibus.eventhandler.steps.AutoRule} is the class used for implementing autorules. Configuration of autorules is handled by the afm_steps.xml file in the WEB-INF/config folder. The autorule methods can be methods in the event handlers, but these are not called by workflow rules.
